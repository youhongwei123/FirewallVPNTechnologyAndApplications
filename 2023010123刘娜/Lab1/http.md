# Lab1：又见面了， HTTP/HTTPS！

## 实验背景

HTTP（HyperText Transfer Protocol，超文本传输协议）是应用层最核心的协议之一。每次打开网页，浏览器与服务器之间就在用 HTTP"对话"。

一次典型的 HTTP 交互分为两部分：

```
浏览器 ──── HTTP 请求 ────▶ 服务器
浏览器 ◀─── HTTP 响应 ──── 服务器
```

**请求报文**结构示例：

```http
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

**响应报文**结构示例：

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

HTTPS 在 HTTP 基础上加入了 TLS 加密，报文内容在传输过程中无法被直接读取。但**浏览器开发者工具**运行在加密之前，可以看到完整的明文请求和响应，是分析 HTTP/HTTPS 协议最方便的入门工具。

1. 用 Chrome 或 Edge 浏览器访问任意 **HTTPS** 站点，例如 `https://www.yxnu.edu.cn/`。

4. 点击列表中第一条请求（通常是页面本身），在右侧查看 **Headers** 标签页，找到 Request Headers 和 Response Headers。
5. 对请求头区域和响应头区域分别**截图**，并按规范命名（见下方截图要求）。

### C. 知识问答

1. HTTP 请求报文由哪几部分构成？请按顺序列出：

   > 答：请求行（方法，URI，版本号），请求头，空行，请求体。

2. 状态码 `404` 代表什么含义？状态码 `500` 和 `503` 有什么区别？

   > 答：404的含义是服务器上没有请求的资源。500表明服务器端在执行请求时发生了错误，也有可能是Web应用存在的bug或某些临时的故障。503表明服务器展示处于超负载或正在进行停机维护，现在无法处理请求。

3. GET 与 POST 方法的主要区别是什么？各适用于什么场景？

   > 答：GET用于获取数据，参数放在URL中，数据量小、不安全、会被浏览器缓存，具有幂等性，适合查询、搜索、页面跳转等场景。POST用于提交数据，参数放在请求体中，数据量大、相对安全、默认不缓存，不具有幂等性，适合登录、注册、提交表单、上传文件等场景。

4. HTTP 与 HTTPS 有什么区别？HTTPS 使用了什么机制来保护数据？

   > 答：1.HTTP 是明文传输，数据容易被窃听、篡改、冒充，默认端口为80；HTTPS 是 HTTP 的加密版本，数据加密传输，更安全，默认端口为443。二者主要区别：HTTP明文传输、不安全、无加密、速度快；HTTPS加密传输、安全、需要证书、速度稍慢。
   2.HTTPS通过SSL/TLS加密机制 保护数据：采用对称加密加密传输数据，用非对称加密安全交换对称密钥，再配合数字证书验证服务器身份，实现数据的机密性、完整性、身份认证。

5. 既然 HTTPS 已经加密，为什么浏览器开发者工具仍然能看到请求和响应的明文内容？

   > 答：浏览器开发者工具能看到明文，是因为它展示的是数据加密前（发送时）和解密后（接收时）在本地内存中的内容。HTTPS是对网络传输过程加密，保护数据在客户端和服务器之间传输时不被窃听、篡改。浏览器开发者工具是本地工具，它查看的是浏览器发送前还未加密、接收后已解密的明文数据。浏览器本身必须持有明文才能正常解析、渲染页面，因此开发者工具可以直接查看明文内容。
