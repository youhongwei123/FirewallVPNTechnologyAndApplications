# Lab2：DNS 初体验，域名是怎么变成 IP 的？

## 实验背景

当我们在浏览器中输入 `www.yxnu.edu.cn` 这样的域名时，计算机并不能直接依靠这个名字找到目标主机，它首先需要通过 **DNS（Domain Name System，域名系统）** 把域名解析成 IP 地址，然后才能继续访问目标服务器。

一次典型的 DNS 解析过程可以简单理解为：

```text
应用程序 ──── 发起域名解析请求 ────▶ DNS 服务器
应用程序 ◀──── 返回域名对应 IP ──── DNS 服务器
```

本实验中，我们使用 C 语言调用系统提供的域名解析接口，查询某个域名对应的 IP 地址；同时使用 Wireshark 抓取 DNS 请求包和响应包，观察域名解析在网络中的实际表现。

> **说明**：本次实验为了降低难度，使用 `gethostbyname()` 完成解析。  
> 这个函数是较老的接口，不支持 IPv6，线程安全性也较差，在实际工程中通常更推荐使用 `getaddrinfo()`。  
> 但在入门阶段，`gethostbyname()` 更容易帮助大家理解“域名解析”这件事是如何发生的。

## 参考代码

将下列代码保存为 `dns_lookup.c`，然后编译运行。

```c
#include <stdio.h>
#include <netdb.h>
#include <arpa/inet.h>

int main()
{
    struct hostent *host = gethostbyname("www.yxnu.edu.cn");
    if (!host)
    {
        herror("查询失败");
        return 1;
    }
    printf("IP: %s\n", inet_ntoa(*(struct in_addr *)host->h_addr));
    return 0;
}

```

编译与运行示例：

```bash
gcc dns_lookup.c -o dns_lookup
./dns_lookup 
```

## 思考题

1. DNS 的作用是什么？为什么访问网站时通常要先进行 DNS 解析？

   > 答：DNS（域名系统）的核心作用是将人类易记的域名（如 www.yxnu.edu.cn）转换为计算机可识别的 IP 地址（如 202.203.86.93），实现域名与 IP 地址的映射。
   访问网站前必须先进行 DNS 解析，是因为网络通信底层依赖 IP 地址来定位主机，域名仅为方便人类记忆而设计，计算机无法直接通过域名建立连接。

2. 你抓到的 DNS 请求包和响应包中的 **Transaction ID** 是否一致？这个字段有什么作用？

   > 答：请求包和响应包的 Transaction ID 完全一致（本次实验中均为 0xc2e8）。
   该字段的作用是唯一标识一次 DNS 查询事务，用于客户端匹配对应的请求与响应，避免多个并发 DNS 查询时出现报文混淆，确保客户端能正确识别服务器返回的解析结果归属。

3. 这次实验中查询类型通常是 `A` 记录。`A` 记录表示什么含义？

   > 答：A 记录（Address Record）是 DNS 中最基础的记录类型，表示将域名映射到对应的 IPv4 地址，是实现域名解析为 IP 地址的核心记录类型。

4. DNS 默认通常使用哪个传输层协议和端口？在什么情况下可能会改用 TCP？

   > 答：DNS 默认使用 UDP 协议，目标端口为 53。
以下情况会改用 TCP 协议：
1.DNS 响应报文长度超过 512 字节（UDP 报文默认最大长度限制）；
2.进行 DNS 区域传输（主从 DNS 服务器之间同步域名数据）；
3.客户端主动发起 TCP 连接请求（如安全校验场景）。

5. 为什么说 `gethostbyname()` 不是现代工程中推荐继续使用的接口？请至少写出两点原因。

   > 答：
1.不支持 IPv6：仅能解析 IPv4 地址，无法兼容 IPv6 网络环境；
2.线程不安全：内部使用静态缓冲区存储结果，多线程并发调用时会导致数据竞争；
3.错误处理简陋：仅能通过 herror() 输出固定错误信息，无法获取更详细的错误码；
4.缺乏可扩展性：不支持自定义超时、重试策略等高级配置。

6. 如果一个域名返回了多个 IP 地址，这通常说明了什么？

   > 答：这通常说明该域名部署了多台服务器，用于实现负载均衡或高可用：
负载均衡：将用户请求分发到不同服务器，避免单台服务器过载，提升访问速度；
高可用：某台服务器故障时，其他服务器仍可提供服务，保证业务连续性；
也可能是基于地理位置的智能解析，让用户连接到最近的服务器节点。
